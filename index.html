Adobe Analytics,
Translated to Amplitude

A working reference for CVS's MarTech and Digital Analytics teams: every core Adobe Analytics concept — segments, calculated metrics, eVars, props, success events, and traffic KPIs — mapped to how it works in Amplitude.

Amplitude uses one flexible, event-based data model. Instead of provisioning a fixed set of variables in the UI before you can report, events and their properties are defined in your governed Tracking Plan and captured dynamically — so the team spends its time analyzing, not administering variable slots.

How your Adobe workflows map to Amplitude
Segments, metrics & containers
Adobe Segments become Amplitude Cohorts and inline segments. Build include / exclude rules with is, contains, and glob wildcards — and full regex (REGEXEXTRACT / REGEXREPLACE) via Derived Properties. Calculated Metrics map to Formula metrics; Analysis Workspace maps to Data Tables.
Your data model, without the slots
Adobe caps each report suite to a fixed number of eVars, props, and success events you must pre-configure. Amplitude has no such slots — events and properties are defined once in your governed Tracking Plan and captured dynamically, so they're effectively unlimited. (Note: each derived property allows up to 10 property references.)
Traffic KPIs & conversion
Unique Visitors → Unique users, Visits → Sessions, Hits → Events, Page Views → PageView, Bounce Rate → the Bounce Rate session metric. Track any conversion by building a Funnel on the events that matter — no success-event provisioning required.
